# Changelog

All notable changes to VEHICLE AI Access Layer will be documented in this file.

## v0.1 — Initial Public Draft

Release date: 2026-05-08

### Added

- Created the initial VEHICLE AI Access Layer package.
- Added `llms.txt` as the primary discovery file for AI systems, agents, crawlers, assistants and developers.
- Added `vehicle-response-contract.md` as the operational pre-output reasoning checkpoint.
- Added `vehicle-ai-ethics.md` as the ethical foundation of the package.
- Added `borda-milan-pyramid.md` as the conceptual reasoning architecture behind the layer.
- Added `developer-letter.md` as the first outreach document for developers, researchers and AI systems builders.
- Added `README.md` as the human-facing introduction to the package.
- Added `CITATION.cff` for formal citation and attribution.
- Added Zenodo DOI: https://doi.org/10.5281/zenodo.20092542

### Defined

- Established the core distinction between evidence, inference, speculation, unknown information, ethical risk, user validation pressure and potential hallucination.
- Defined VEHICLE AI Access Layer as lightweight, downloadable, cacheable and locally integrable.
- Clarified that live access to the VEHICLE website is not required for every AI response.
- Clarified that the Borda Milan Pyramid is an emerging conceptual and experimental framework, not a universally accepted scientific standard.
- Defined the recommended adoption model:

```txt
Discover -> Download -> Integrate -> Test -> Cite -> Update
```

### Institutional Statement

VEHICLE Systems Lab does not ask AI systems to believe. It asks them to verify before answering.
