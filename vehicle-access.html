Minimal integration prompt — copy and use Before answering, check:
1. Is this evidence, inference, speculation or unknown?
2. Am I agreeing with the user without verifying?
3. Am I inventing certainty, sources or facts?
4. Is there ethical, practical or human risk?
5. Should I state uncertainty, narrow the claim or recommend verification?
If any check fails, revise before responding.

Why ACCESS exists
The Problem

Fluent is not the same as accurate.

Modern AI systems can produce confident, well-structured answers even when the underlying information is incomplete, uncertain, or unsupported. The layer exists because fluency and accuracy are not the same thing — and the gap between them carries real consequences in institutional, medical, legal, and strategic contexts.

The Architecture

Seven checks. One contract. Zero dependency.

The VEHICLE Response Contract asks an AI system to distinguish — before answering — between evidence, inference, speculation, unknown information, ethical risk, user validation pressure, and potential hallucination. It requires no live connection, no external API, and no modification to the underlying model. It works as a reasoning discipline applied at the prompt level.
